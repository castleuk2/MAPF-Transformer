# MAPF Transformer Policy

A reference implementation of the proposed hierarchical spatio-temporal MAPF
policy. It keeps the public policy wrapper close to MAPF-GPT (`act`,
`act_batch`, `reset_states`) while using a new structured I/O design.

## Architecture

- Persistent Ego-centered **15×15 static spatial memory**.
- Successful movement updates only the incoming row/column (**15 cells**);
  WAIT or failed movement reuses the map representation.
- 225 one-cell tokens are compressed by cell-to-latent cross-attention into a
  configurable number of map latents; the current comparison fixes this at
  **32 learned map latent tokens**.
- Ego + up to 24 neighbors are represented as 25 stable local entities.
- Every entity starts from **8 typed field tokens**: stable local slot, local X,
  local Y, directional one-hop CTG (`UP/DOWN/LEFT/RIGHT`), and exact goal
  distance. A shared agent-local encoder reduces those fields to one entity
  token without mixing different agents.
- The 25 entity tokens query map latents through agent-to-map cross-attention.
- Validity is an attention/padding mask rather than a field token. Track reset
  is a structural embedding broadcast over that agent's eight fields.
- 25 agent tokens + one transition token = **26 tokens/frame**.
- There is no separate pre-temporal agent-set attention. The Temporal
  Transformer performs bidirectional attention within each frame and causal
  attention across frames.
- History 5 uses `5×26+1=131` tokens; History 10 uses `10×26+1=261`, including
  one final `[ACT]` query.
- The action head follows POGEMA order: `WAIT, UP, DOWN, LEFT, RIGHT`.

The model is a hierarchical spatial encoder + temporal policy Transformer. It is
not a standard text GPT and not a conventional encoder-decoder Transformer.

## Dataset format

Each `.npz` episode contains:

- `obstacles`: `[H,W]`, uint8, 0 free / 1 blocked.
- `positions`: `[T+1,N,2]`, int16.
- `goals`: `[N,2]` or `[T+1,N,2]`, int16.
- `actions`: `[T,N]`, uint8 in POGEMA action order.
- `arrival_steps`: `[N]`, each agent's final stable goal-arrival step. The
  dataset exposes all pre-arrival actions plus the configured fraction of
  final stable-goal WAIT targets.
- optional `neighbor_ids`, `neighbor_valid`, `track_reset` with shape
  `[T+1,N,K]`.
- `neighbor_ids_24`, `neighbor_valid_24`, `track_reset_24` with shape
  `[T+1,N,24]`, precomputed from the full episode using the same stable-slot
  policy as online inference.

A JSONL manifest points to episode files. One supervised item is
`(episode, ego agent, time step)`. The loader constructs the configured latest
5 or 10 frames; missing initial history is represented by invalid PAD frames,
so no separate "initial-history" dataset is required. Random-history configs
deterministically assign each training sample a retained length between one and
the available upper bound. The full-history config uses the naturally available
1, 2, 3, ... frames up to its upper bound, matching online inference.

Learned map latents and embeddings are never serialized in the dataset. They
are generated by the current model, preserving the ability to change token
encoders without regenerating expert trajectories.

## Training

The default embedding width is 256. The exact parameter count depends on the
map-latent, agent-local, and temporal-layer configuration.


```bash
python train.py --config configs/mapf_transformer_base.yaml
```

For the one-hour dataset, use `torchrun` with the configured model.
Run from the workspace root. One GPU:

```bash
torchrun --standalone --nproc_per_node=1 mapf-transformer-policy/train.py \
  --config mapf-transformer-policy/configs/mapf_transformer_base_1h.yaml
```

Two GPUs:

```bash
torchrun --standalone --nproc_per_node=2 mapf-transformer-policy/train.py \
  --config mapf-transformer-policy/configs/mapf_transformer_base_1h.yaml
```

`gradient_accumulation_steps` is a global value and is divided by DDP world
size. Both commands therefore keep the same effective batch and optimizer-step
schedule.

For an interactive foreground run that is also recorded, pipe both output
streams through `tee`:

```bash
torchrun --standalone --nproc_per_node=2 mapf-transformer-policy/train.py \
  --config mapf-transformer-policy/configs/mapf_transformer_base_1h.yaml \
  2>&1 | tee mapf-transformer-policy/runs/mapf_lns2_1h/console.log
```

Rank 0 also writes structured train/validation/checkpoint events to
`metrics.jsonl` automatically.

### Agent-25 temporal-history comparison

Every experiment uses 32 map latents and exactly 25 physical entity tokens
(24 stable neighbor slots plus Ego). For each entity, one learned query
cross-attends to eight typed metadata tokens and produces one agent token.
Each agent then queries the map latents. There is no interaction latent,
pre-temporal agent-set self-attention, or graph-attention branch: the Temporal
Transformer performs both same-frame bidirectional attention and cross-frame
causal attention.

Before training a 24-neighbor model, add full-episode 24-slot tracking to the
existing NPZ files. The original 15-slot arrays are preserved for legacy
checkpoints, and the additional arrays are written atomically:

```bash
PYTHONPATH=mapf-transformer-policy/src \
python mapf-transformer-policy/upgrade_neighbor_tracking.py \
  --manifest pogema-mapf-transformer/data/mapf_lns2_1h/train_manifest.jsonl \
  --manifest pogema-mapf-transformer/data/mapf_lns2_1h/val_manifest.jsonl \
  --workers 24 \
  --overwrite
```

The 24-neighbor loader then uses `neighbor_ids_24`, `neighbor_valid_24`, and
`track_reset_24` computed from the complete episode, so Stable Slot identity
matches the online runtime. When all slots are occupied, a newly visible agent
replaces the farthest tracked agent only if it is strictly closer to Ego.
`--overwrite` ensures that NPZ files produced by an older slot policy are
updated to this replacement policy.

These runs use `distance_encoding: exact` with 64 embeddings:
values 0 through 62 are exact BFS steps and 63 is reserved for unreachable.
The current train/validation trajectories have a maximum observed BFS input
distance of 62. Existing checkpoints without this config field keep the
legacy 64-value four-hop bucket encoding.

- `agent25_map32_history5_random.yaml`: random 1..5-frame history augmentation;
  context upper bound `5 * 26 + 1 = 131`.
- `agent25_map32_history10_random.yaml`: random 1..10-frame history
  augmentation; context upper bound `10 * 26 + 1 = 261`.
- `agent25_map32_history5_full.yaml`: no artificial truncation. It uses the
  naturally available 1, 2, 3, 4, then 5 frames exactly as online inference.

Run a configuration on two GPUs:

```bash
CONFIG=agent25_map32_history5_random
RUN=agent25_map32_history5_random
mkdir -p "mapf-transformer-policy/runs/${RUN}"
set -o pipefail
CUDA_VISIBLE_DEVICES=0,1 \
PYTHONPATH=mapf-transformer-policy/src \
torchrun --standalone --nproc_per_node=2 \
  mapf-transformer-policy/train.py \
  --config "mapf-transformer-policy/configs/${CONFIG}.yaml" \
  2>&1 | tee "mapf-transformer-policy/runs/${RUN}/console.log"
```

Use three machines for `history5_random`, `history10_random`, and
`history5_full`. Compare history 5 versus 10 using the two random-history runs,
then compare random truncation versus inference-aligned history using the two
history-5 runs.

Important outputs:

- `resolved_config.yaml`
- `best.pt` when validation is enabled
- periodic `step_*.pt`

## Online inference interface

```python
from mapf_transformer import (
    MAPFTransformerInference,
    MAPFTransformerInferenceConfig,
)

policy = MAPFTransformerInference(
    MAPFTransformerInferenceConfig(checkpoint_path="runs/model/last.pt")
)
actions = policy.act(observations)
policy.reset_states()
```

`observations` must expose `global_obstacles`, `global_xy`, and
`global_target_xy`, matching POGEMA's MAPF observation mode. The companion
project provides the environment adapter and dataset generator.

## Design notes

- The map buffer is updated from **actual displacement**, never merely from the
  commanded action.
- Directional one-hop CTG categorizes whether each movement decreases, preserves,
  or increases cost-to-go, or is blocked/unreachable. It is an input feature,
  not an output hard mask.
- When there are fewer than 24 visible neighbors, invalid stable slots retain
  the fixed tensor layout but are suppressed by the validity mask.
- Map reconstruction is an optional training-only auxiliary loss that helps the
  16 latents preserve narrow passages and obstacle boundaries.
